# 设计审批模板（表单 + 流程）

## 入口
- 「应用」→「审批」→「模板管理」（仅管理员可见）

## 模板列表能做什么
- 按名称搜索、按启用 / 停用状态筛选
- 新建模板、编辑模板（新建页还可一键「模板导入」批量导入内置预设，见下）
- 启用 / 停用模板（停用后用户无法再用它发起，已发起的不受影响）
- 删除模板（点删除后有二次确认）

## 新建 / 编辑模板
进入模板设计器，配置以下内容：
1. **基础信息**：模板名称、图标、所属分类（假勤 / 行政 / 财务 / 人事 / 其他）
2. **表单设计**：用表单设计器拖拽添加字段，设置标签、是否必填、选项、校验规则等。支持 14 种字段类型，详见 [[approve.form-field.concept]]
3. **流程设计**：用流程设计器搭审批流程

## 模板导入（从内置预设批量建）
- 仅「新建模板」页有：选项卡末尾的「模板导入」按钮，点开是弹窗（不切换选项卡）
- 弹窗按分类列出全部内置预设模板，可自由多选、按分类「全选 / 全不选」
- 点「导入」直接批量建好这些模板并跳回模板列表（不是写入当前正在编辑的表单）
- 模板名按当前界面语言入库（英文界面建英文名，中文界面保持中文）
- 与现有模板同名的预设标「已存在」，仍可勾选；导入前会二次确认是否产生重复
- 用途：迁移过旧数据的环境首启不再播种、缺少系统默认模板时，可用它补齐

## 流程设计器
- 从发起节点开始，向下逐个添加节点：**审批节点**、**抄送节点**、**条件分支**
- 审批节点配置：节点名、审批人来源（指定成员 / 部门主管按层级 / 发起人本人）、多人时的审批方式（或签 / 会签 / 依次）
- 条件分支配置：按表单的数字 / 金额 / 单选字段设条件，把流程导向不同后续，必须留一条默认分支，详见 [[approve.route.concept]]
- 保存前会校验流程合法（至少要有审批节点、审批人不为空、分支要有默认分支等）

## 保存
设计完点保存即生效，用户在「发起申请」里立即能选到（前提是模板为启用状态）。
