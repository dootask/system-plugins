# 审批单状态与流转

## 定义
审批单（实例）是一次具体的申请。每发起一次就生成一张，按模板定义的流程逐节点流转，状态随之变化。

## 状态一览
- **审批中**：正在流转，有人在处理或等待处理
- **待修改 / 退回中**：被审批人退回给发起人，发起人改完重新提交才继续，详见 [[approve.return-resubmit.howto]]
- **已通过**：所有节点走完且都批准，终态
- **已拒绝**：某审批人拒绝，整单终止，终态
- **已撤回**：发起人在无人处理前主动撤回，终态，详见 [[approve.withdraw.howto]]

## 在详情页能看到什么
打开一张审批单，从上到下能看到：
- **基本信息**：标题、当前状态、发起人、发起时间
- **表单内容**：发起时填的内容（只读；被退回时发起人可在此编辑后重新提交）
- **流程进度**：竖向步骤条，显示每个节点的名称、审批方式、参与人及各自的处理状态（待处理 / 已通过 / 已拒绝 / 已退回等），加签人会标「加签」
- **操作区**：按你的角色显示可用按钮（批准 / 拒绝 / 退回 / 转交 / 加签 / 撤回 / 仅评论）
- **时间线**：按时间倒序的全部动作记录（提交、审批意见、评论、退回、转交、加签、撤回等），含上传的图片缩略图

## 谁能看到一张审批单
发起人、所有历史参与人（审批人 / 抄送人 / 加签人）以及管理员可见，其他人无权查看。
